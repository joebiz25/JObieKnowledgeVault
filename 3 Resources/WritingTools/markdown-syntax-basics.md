
## Overview

Markdown is very similar to hypertext markup language.
It seems like there is a intentionality in calling in markdown in reference to markup in html
Just like any computer language, markdown has its translator, called processor (like compiler for C and interpreter for python)
There can be minor variances between markdown processors to be aware of
Nearly all markdown applications support the basic syntax as outlined in the original Markdown design

## Syntax

# Headings

Use the '#' symbol at the beginning of a statement to make the statement a heading. The number of the '#'s used should correspond to the level of the heading:

    # Heading Level 1       (html <h1>Heading Level 1</h1> )

    ## Heading Level 2      (html <h2>Heading Level 2</h2>)

    ### Heading Level 3      (html <h3>Heading Level 3</h3>)

Alternatetely, using any number of '=' or '-' signs under the text also makes the text a heading level 1 and heading level 2 respectively. This method, of course, will not work for more than 2 levels of headings

Best practice in to separate the heading text from '#' with space and also to generally leave an empty line before and after heading

# Paragraphs

Use a blank line to separate the line(s) of text forming the paragraph from other text
In contrast html uses "<p> </p>" markers.

Best practice is NOT to indent paragraphs with tab or space except the paragraph is a list 

# Line Breaks

Use a trailing white space (2 spaces) to create a line break.
Don't use more than 2 spaces for compatibility reasons

Alternatively, if the Markdown application supports HTML, "<br>" can also be used 

# Emphasis

**Bold**: use 2 asterisks before and after text to bold

__Bold__: 2 underscores can also be used

*Italize*: use an asterisk before and after the text to be italized

_Italize_: use an underscore before and after the text to be italized

***Bold and Italize***: use 3 asterisks before and after the text to be bolded and italized

___Bold and Italize___: use 3 underscores before and after the text to be bolded and italized
 
Best practice is to be consistent and use only asterisks since the treatment of underscore(s) in the middle of a word is not standardly agreed upon in all markdown applications
